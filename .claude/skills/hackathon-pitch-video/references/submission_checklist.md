# Pre-submission checklist — the gate before you upload

Run every item. A great project loses to a video that's 40 seconds too long, has no audio in the
first take, or links to a private repo. Don't skip the boring ones.

## Format & rules compliance
- [ ] **Length** is at or under the hackathon's cap (re-read the rules — caps vary 2-5 min).
- [ ] **File format / resolution** matches requirements (usually MP4 H.264, 1080p).
- [ ] **Hosting/platform** is correct (YouTube unlisted? Devpost upload? Drive link with
      view access?). Confirm the exact place and method.
- [ ] **Link is public / view-accessible** — open it in an incognito window to verify. (#1 silent
      failure: private/“request access” links.)
- [ ] Any required **on-screen elements** (team name, project name, sponsor logos) are present.

## The first 10 seconds
- [ ] The **one-liner lands in the first 10 seconds** — a judge who stops at 0:10 still knows what
      this is.
- [ ] No slow intro / agenda / "today we'll present..." eating the hook.
- [ ] Audio is present and clean from the very first second (check — first takes often clip/cut).

## Rubric coverage (cross-check `judging_rubric.md`)
- [ ] **Innovation** — one clear sentence names the thing only this project does.
- [ ] **Technical Execution** — the video *shows the thing running*, not just slides.
- [ ] **Design/UX** — demo is legible (zoomed, big fonts) AND the video itself is clean.
- [ ] **Theme/Impact** — the challenge theme is named in the judges' own words + who it helps.
- [ ] Every scored criterion has a moment that earns it; time is weighted toward the weights.

## Demo quality
- [ ] Demo shows **one real task end to end**, not a UI tour.
- [ ] Everything shown **actually works** — no narrating vaporware as if it's live.
- [ ] Screen text is readable on a phone (zoomed ≥150%, no tiny fonts).
- [ ] No dead air / long loads / visible errors / typos in the demo.
- [ ] Notifications, extra tabs, and desktop clutter are hidden.

## Production quality
- [ ] Audio is consistent volume, no clipping, no loud background noise.
- [ ] **Captions/subtitles** present and correct (esp. product/jargon words).
- [ ] Key metric / result is on screen near the demo, big enough to read.
- [ ] **Repo / live link** is on the closing frame and readable (and actually works).
- [ ] Watched start-to-finish once, in full, on the final export.

## Honesty & credibility
- [ ] Limitations are stated honestly (reads as rigor; judges reward it).
- [ ] No claims the project can't back up if a judge opens the repo.
- [ ] Team / contributors credited as required.

## Final
- [ ] Title + description fields filled (some platforms judge these).
- [ ] Submitted **before** the deadline with buffer — uploads and processing take time.
- [ ] Submission confirmation received / screenshotted.
